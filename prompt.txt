**User**: Create a lights script that lets the user customize when lights turn on/off. Default should look up location and time, adjusting to civil sunset/sunrise. No light sensor yet.

**Claude**: Created `src/salmoncv/lights.py` — a long-running lights scheduler:
- Defaults to Quinhagak, AK coords (59.75°N, 161.92°W) using the `astral` library
- Calculates civil twilight daily: lights ON at dusk, OFF at dawn
- Manual override: `--on-time 20:00 --off-time 06:30`
- Can override just one: `--on-time 21:00` (keeps auto dawn for off)
- Handles midnight sun and polar night (lights stay in current state)
- `--dry-run` mode prints schedule without toggling GPIO
- `--lat`, `--lon`, `--timezone` for other deployment locations
- Safety: lights always turned off on Ctrl+C exit

Usage:
- Auto mode: `salmoncv-lights`
- Custom: `salmoncv-lights --on-time 20:00 --off-time 06:30`
- Test: `salmoncv-lights --dry-run`

**User**: Lights command printed "Lights ON" but the lights didn't actually turn on.

**Claude**: The issue was `gpiozero` — it resets GPIO pins when the process exits, so the relay turned on for a split second then immediately turned off. Rewrote `power.py` to use `pinctrl` (built into Pi 5) which sets GPIO state permanently — it persists after the process exits. Also updated `lights.py` to reuse `lights_on()`/`lights_off()` from `power.py` instead of managing GPIO directly. Removed `gpiozero` from project dependencies.

**User**: Still not working — lights clicked on late and "lights off" didn't turn them off.

**Claude**: Relay polarity was inverted. Testing with `pinctrl set 17 op dh` confirmed the relay is active-high, not active-low. Fixed `power.py`: ON = drive high, OFF = drive low.
